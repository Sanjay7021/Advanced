Build a banking application, which will have deposit and withdraw feature 

There will be two pages - Customer Portal and Saving Account form

Customer Portal
-------------------------
The control IDS and Field names (proper validation will be implemented)

All four fields are input fields with type "text"

1. Customer ID - customerID (error control ID - IDerr)

    If customerID empty = error message = Please enter Customer ID
    if any alphabet = error message = Invalid customer ID. Please enter a valid number.
    if any special character = Invalid customer ID. Please enter a valid number.

2. Account Number - accNum (error control ID - accErr)
    
    If acc empty = error message = Please enter Account Number
    if any alphabet = error message = Please enter correct account Number
    Account number should 14 digits long 

    example: 
    invalid - 12345679
    valid - 63580201001476 



3. Customer Name - customerName (error control ID - NameErr)

   If Customer Name empty = error message = Please enter Customer name
    if any number = error message = Invalid customer name. Name should not contain spaces at the left or right.
    if any special character = Invalid customer name. Name should not contain spaces at the left or right.

4. Account Balance - accBal (error control ID - balErr)

    Account bal. minimum 5000 validation


    if any alphabet - Please enter only number
    if amount less than 5000 - Amount should be minimum 5000

Two Buttons:

5. Submit button - control ID = customerForm
6. Redirection button - control ID =  redirect

Task:

- Store customer data
- implement routing to navigate to Saving form page
- implement proper validation 



Saving account page
----------------------

Fields and Control IDs, proper validation for Radio fields and amount field

1. Customer ID dropdown - customerSelect
2. Customer Name - savingAccName (input - text)
3. Customer Account Number - savingAccountNumber (input - text)
4. Radio Buttons (Withdraw and Deposit):
    - Withdraw (depositRadio): A radio button for selecting the withdrawal option.
    - Deposit (withdrawRadio): A radio button for selecting the deposit option.

    radio button required 
    error messages: Please select one option

5. Amount (Input field) - amount, (error ID - amountErr)

    amount required - error message = Please mention amount for transaction

6. Empty <p> tag to show total balance - accBalP
7. Submit Button - checkTransaction


if amount is greater than total balance. show one message - Insufficient funds (control ID = accountFundError)

Task:
- By selecting the Customer ID from the dropdown, populate the customer name and account number in their respective fields.
- Display the customer's account balance in the `<p>` tag.
- Implement logic for withdrawal and deposit actions, updating the total balance accordingly.
- persist the customer data during refresh of page
